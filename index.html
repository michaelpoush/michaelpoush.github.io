Comprendre les métriques

Ce que révèlent ces scores

Mobile catastrophique, desktop acceptable — un écart qui coûte cher

LCP 20,2 secondes mobile
RECORD NÉGATIF
✗ 8× le seuil
Le LCP mesure le temps avant que le contenu principal apparaisse. À 20,2 secondes, c'est le score le plus faible qu'on rencontre. Le seuil "acceptable" est 4s, le seuil "bon" est 2,5s.
Impact : Google indique que 53% des visiteurs mobiles quittent une page après 3 secondes. À 20 secondes, le site perd probablement 90%+ de ses visiteurs mobiles. Avec 60%+ du trafic web qui vient du mobile, c'est une hémorragie.
📺C'est comme une télévision qui met 20 secondes à s'allumer après avoir appuyé sur la télécommande. La grande majorité des gens appuient une deuxième fois sur le bouton — et partent.
CLS 0,403 mobile — mise en page instable
CUMULATIVE LAYOUT SHIFT
✗ 13× le seuil
Le CLS mesure combien les éléments de la page "bougent" pendant le chargement. À 0,403, les boutons et textes se déplacent massivement. Seuil acceptable : 0,1.
Impact concret : L'utilisateur clique sur un bouton — mais juste avant, la page bouge et il clique ailleurs. C'est la source principale de frustration UX sur mobile. Google pénalise directement ce score dans son ranking.
📄Imaginez lire un document et que les paragraphes se déplacent pendant que vous lisez. C'est l'expérience d'un visiteur mobile sur almadius.com.
Desktop correct — mobile désastreux
PARADOXE MOBILE/DESKTOP
⚠ Écart majeur
Sur desktop, les scores sont acceptables (LCP 4,6s, CLS 0,009). C'est souvent le signe que le site a été conçu et testé uniquement sur ordinateur, sans optimisation mobile spécifique.
Cause probable : Vidéo en hero section non optimisée pour mobile, images lourdes non adaptées, scripts tiers qui bloquent le rendu. Ces problèmes sont corrigeables sans refonte complète.
Score SEO & Bonnes pratiques — 100
POINTS FORTS
✓ Parfaits
Malgré les performances catastrophiques sur mobile, la structure technique SEO est irréprochable. Les balises, HTTPS et bonnes pratiques web sont parfaits.
Bonne nouvelle : Les fondations sont saines. Le problème est uniquement lié à la vitesse et à la stabilité du chargement mobile — ce qui est corrigeable sans refonte.
💡
Priorité absolue : optimisation mobile
Remplacer la vidéo hero par une image optimisée, passer toutes les images en WebP avec lazy loading, et différer les scripts tiers. Ces 3 actions seules pourraient faire passer le LCP de 20s à sous 4s et éliminer le CLS en 2-3 semaines de travail.
